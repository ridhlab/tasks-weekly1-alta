Section 1 - Version Control

Pengertian

Version Control System, atau biasa disingkat VCS, adalah sebuah sistem yang membantu untuk mentrack perubahan suatu dokumen dari waktu ke waktu. Dalam penggunaannya, VCS banyak digunakan oleh para developer ketika sedang berkontribusi dalam sebuah pengembangan perangkat lunak. Contoh VCS yang biasa digunakan oleh para developer adalah git.

Dalam pengoperasiannya, git sering kali diintergrasikan dengan Github. Github adalah sebuah layanan berbasis web yang digunakan oleh para developer untuk memudahkan dalam berkontribusi sebuah projek. Jika git hanya bisa digunakan di local komputer kita, maka github adalah penghubung antara git yang ada di masing-masing komputer developer. Dengan adanya github, kita sebagai developer dapat berkolaborasi secara realtime, mengecek dan mentrack perubahan pada kode program, dan memanajemen suatu projek perangkat lunak. Karena keunggulannya itu, github sangat popular di dunia open source. Sehingga tak ayal, github sering dijuluki sosial medianya para programmer.

Workflow

Ketika menggunakan git, kita biasanya dihadapkan dengan tiga keadaan. Git menyebutnya sebagai workflow. Penting untuk mengingat dan memahami konsep dari 3 workflow itu, supaya memudahkan kita dalam berkolaborasi. Tiga keadaan itu adalah : commited, modified dan staged. Commited adalah ketika data perubahan sudah disimpan di local git kita. Modified adalah ketika kita memodifikasi beberapa file, dan belum disimpan di local git kita. Dalam programming, biasanya yang masuk dalam workflow modified yaitu perubahan kode program, atau perubahan nama file. Staged adalah kondisi kita sudah menambahkan data yang diubah ke dalam versi saat ini, tapi belum dicommit.

Hal yang tak kalah penting untuk dipahami selanjutnya adalah tentang area atau bagian yang ada di dalam git, yaitu working directory, staging area, dan git directory. Working directory adalah directory yang sedang dikerjakan, namun belum masuk ke staging area. Biasanya ketika kita memodifikasi file, maka kita sedang melakukannya di working directory. Staging area adalah sebuah tempat untuk menampung data apa saja yang sudah diubah tapi masih belum masuk ke tahap commit. Sehingga perubahannya masih belum di track oleh local git. Git directory adalah tempat yang berisi file-file yang sudah tersimpan perubahannya.

Perintah Dasar Git

Beberapa perintah yang sering digunakan antara lain :

  1. Git init

    2. Perintah ini digunakan untuk membuat sebuah folder di local computer kita menjadi sebuah repository. Ketika folder tersebut menjadi repository, maka akan ada folder .git secara hidden yang berisi informasi mengenai repository tersebut. Syntaxnya adalah git init.

  2. Git clone

    3. Perintah ini digunakan untuk menclone sebuah repository dari github ke dalam komputer kita. Git clone biasanya perintah yang pertama kali kita gunakan ketika pertama kali ingin berkontribusi dalam sebuah projek yang sudah ada di github. Syntaxnya adalah git clone <url>.

  3. Git remote

    4. Git remote adalah perintah untuk menghubungkan repository yang ada dalam github. Dalam prakteknya perintah yang biasanya dipake yaitu git remote add <name> <url> untuk menambah daftar remote repository, dan git remote rm <name> <url> untuk menghapus remote repository.

  4. Git status

    5. Perintah ini digunakan untuk mengetahui status dari file, working tree, dan branch yang sedang aktif. Syntaxnya adalah git status.

  5. Git branch

    6. Git branch digunakan untuk melihat branch apa saja yang ada di repository kita. Cukup dengan git branch, maka branch yang terdaftar akan muncul. Untuk menambah branch baru, cukup menuliskan syntax git branch <name>.

  6. Git pull

    7. Sesuai dengan namanya, git pull digunakan untuk menarik remote repository ke local repository kita. Ini akan membuat local repository akan berisi file yang ada di remote repository. Syntaxnya git pull <remote-name> <branch>.

  7. Git add

    8. git add <filename> berfungsi untuk menambahkan file ke dalam staging area sehingga siap untuk di commit. Untuk menambahkan semua file yang telah terdeteksi perubahannya, maka tuliskan perintah git add ..

  8. Git commit

    9. Git commit digunakan untuk menyimpan perubahan file ke dalam git directory. Git commit bekerja dengan memindahkan file-file yang ada di staging area ke git directory. Syntaxnya yaitu git commit –m “message”.

  9. Git push

    10. Perintah git push <remote-name> <branch> akan memperbarui remote repository branch target di github sama dengan yang ada di local repository kita yang telah tersimpan.
